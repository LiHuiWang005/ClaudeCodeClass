概念理解

什么是漏斗分析

漏斗分析是一种分析客户在多阶段流程中逐步流失与转化的方法。

  • 每一层代表业务流程中的一个关键步骤。
  • 每一层的数量表示仍然留在流程中的客户数。
  • 通过比较相邻阶段，可以发现哪里流失最多、哪里需要优化。
讲解说明：可以把它理解为“很多潜在客户进入流程，经过拜访、报价、谈判后，最后只有一部分签约”。重点不是画图，而是发现业务改进点。

应用场景

漏斗分析适合哪些业务

销售转化

线索 → 拜访 → 报价 → 成交

用于提升成交率，判断客户在哪一步离开。

售后服务

报修 → 派工 → 到场 → 完成

用于缩短服务周期，发现响应或处理瓶颈。

设备维保

设备清单 → 客户沟通 → 方案 → 合同

用于提升续约率和维保合同转化效率。

项目跟进

需求确认 → 方案设计 → 报价 → 签约

用于识别方案、预算、决策链条中的阻塞点。

讲解说明：只要业务有明确步骤，就可以使用漏斗分析。暖通空调行业中的售后维保、续约销售、报价转化和项目推进都很适合。

核心指标

转化率、流失率和总体转化率

阶段转化率

下一阶段客户数 ÷ 当前阶段客户数 × 100%

阶段流失率

1 - 阶段转化率

流失客户数

当前阶段客户数 - 下一阶段客户数

总体转化率

最终签约客户数 ÷ 初始目标客户数 × 100%

示例

客户拜访 80 家，提交报价 50 家。

阶段转化率 = 50 ÷ 80 = 62.5%

流失率 = 1 - 62.5% = 37.5%

流失客户数 = 80 - 50 = 30 家

讲解说明：转化率回答“有多少客户进入下一步”，流失率回答“有多少客户没有继续走下去”。两个指标一起看，才能判断流程效率。

图表解读

漏斗图的构成和解读

漏斗图通常包含

  • 阶段名称，例如目标客户、客户拜访、提交报价。
  • 每阶段客户数量，用于判断规模变化。
  • 阶段转化率或占比，用于判断推进效率。
  • 流失数量，用于定位损失最大的环节。
  • 异常标记，用于提示重点关注位置。

案例分析过程

从业务问题到数据分析步骤

1. 明确业务目标

提升维保合同签约率，找出客户在哪些阶段流失。

2. 确认漏斗阶段

目标客户 → 客户拜访 → 提交报价 → 技术交流 → 商务谈判 → 签订合同。

3. 整理客户数量

记录每个阶段剩余客户数，保证阶段口径一致。

4. 计算关键指标

计算转化率、流失率、流失客户数和总体转化率。

5. 找出流失环节

同时看绝对流失客户数和阶段流失率。

6. 输出优化建议

结合销售跟进、技术方案、客户预算和决策链分析原因。

讲解说明：漏斗分析要先定义阶段顺序，否则计算没有意义。AI 可以帮助整理表格和生成初稿，但阶段口径和业务解释必须由人来确认。

问题诊断

问题发现与原因分析

问题环节 数据表现 可能原因
客户拜访 → 提交报价 流失30家 客户需求不明确、预算不足、拜访后跟进不及时、设备资料不完整。
技术交流 → 商务谈判 流失率50.0% 技术方案价值没有讲清，节能收益未量化，决策人未参与，竞争对手介入。
商务谈判 → 签订合同 流失率46.7% 价格异议、合同条款分歧、付款周期较长、客户审批流程复杂。
讲解说明：数据只能指出“哪里有问题”，不能自动说明“为什么”。原因分析要结合现场沟通、客户类型、设备状态、报价策略和竞争环境。

改进动作

围绕关键流失环节提出优化建议

环节 优化建议 预期效果
拜访后未报价设计标准需求确认表；拜访后48小时内输出初步方案。提高报价推进率。
技术交流流失增加节能测算、故障风险、停机损失说明。提高客户认可度。
商务谈判流失设置基础版、标准版、保障版三档维保套餐。降低价格沟通难度。
决策链复杂识别使用部门、设备部门、采购部门和财务部门。提高推进效率。
跟进不连续建立客户跟进台账和下次行动日期。减少沉默客户。
讲解说明：优化建议要具体，不能只写“加强沟通”。每条建议都要对应一个数据问题，最好能落到表单、时限、话术、方案或管理机制。
