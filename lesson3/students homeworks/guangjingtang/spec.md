# 复利学习页面 — 产品规格书 (spec.md)

## 一、概述

### 1.1 产品目标
做一个**多角度、深入全面**的"复利"学习页面，让用户通过图文、交互程序、外部资源，透彻理解复利的概念、公式、力量与陷阱。

### 1.2 目标用户
- 理财入门者：想理解"钱生钱"的基本原理
- 学生：需要直观感受指数增长的威力
- 任何对个人财务感兴趣的人

### 1.3 核心体验
用户进入页面后，沿预设叙事线纵向滚动，依次经历"是什么 → 怎么算 → 有多强 → 反面案例 → 启发思考"的完整学习旅程。

---

## 二、信息架构

### 2.1 叙事线 (9 个模块，单页纵向滚动)

```
Hero（首屏）
  ↓
Definition（定义：什么是复利）
  ↓
Intuition（直觉培养：指数增长的威力）
  ↓
Calculator（核心：可交互复利计算器）
  ↓
TimePower（时间的力量：早开始 vs 晚开始）
  ↓
Rule72（72 法则：翻倍年限估算）
  ↓
DarkSide（债务陷阱：复利的反面）
  ↓
Quotes（名人名言轮播）
  ↓
FurtherReading（延伸阅读资源）
```

### 2.2 为什么是这个顺序
1. **先认知** — 定义、直觉，建立心智模型
2. **再动手** — 计算器亲自算，产生"噢，原来如此"
3. **再深化** — 时间力量、72 法则，多角度巩固
4. **警示意** — 债务陷阱，展示复利的双刃剑本质
5. **收尾** — 名言启发 + 延伸资源，鼓励继续学习

## 三、模块详细设计

### 3.1 Hero（首屏区）
- **内容**: 标题"理解复利"、副标题一句话概括复利的意义、向下滚动提示
- **视觉**: 简洁大气，可能配合粒子/生长动画暗示"增长"
- **状态**: 纯静态文案

### 3.2 Definition（定义区）
- **内容**: 1-2 段说明什么是复利，核心公式 `A = P(1 + r/n)^(nt)` + 每变量标注
- **视觉**: 公式大卡片突出展示，变量用不同颜色标注
- **状态**: 静态文案 + 公式渲染

### 3.3 Intuition（直觉培养）
- **内容**: 一个固定例子的指数增长可视化——比如"1 元每天翻倍，30 天后是多少？"（答案 5.37 亿）
- **视觉**: 逐步增长的柱状图或数字滚动
- **状态**: 自动播放动画 / 用户触发放大可

### 3.4 Calculator（复利计算器）★核心
- **输入参数**:
  - 本金 (P)：初始投入金额
  - 年利率 (r)：%
  - 年限 (t)：投资年数
  - 复利频率 (n)：年/月/日/连续
  - 定期追加 (可选)：每期追加金额
- **输出结果**:
  - 终值 (A)
  - 累计利息
  - 本金占比 vs 利息占比
- **图表**: 折线图对比（本金线 / 单利线 / 复利线）
- **交互**: 输入即实时更新，数字 count-up 动画
- **校验**: 异常输入（负值、过大值）给出提示

### 3.5 TimePower（时间的力量）
- **场景**: 小明（20 岁开始，投 10 年）vs 小红（30 岁开始，投 35 年）
- **可视化**: 双角色折线图叠加，标注交叉点
- **关键洞察**: 自动生成对比文案（如"早开始 10 年，最终多攒 X 万"）
- **交互**: 可调整两人起始年龄与月供

### 3.6 Rule72（72 法则）
- **内容**: 解释 72 法则——用 72 除以年利率 ≈ 资金翻倍所需年数
- **交互**: 输入利率 → 显示估算 vs 精确计算结果
- **可视化**: 曲线上动态打点标注翻倍点

### 3.7 DarkSide（债务陷阱）
- **场景**: 信用卡欠款，只还最低还款额的还款年限模拟
- **输入**: 欠款本金、年化利率、最低还款比例
- **输出**: 还清年限、累计利息
- **视觉**: 警示色柱状图展示利息膨胀
- **对比**: 与 Calculator 区的"投资复利"形成镜像对照

### 3.8 Quotes（名人名言）
- **内容**: 5-8 条复利/投资相关名言（巴菲特、爱因斯坦等）
- **交互**: 卡片自动轮播 + 手动左右切换
- **细节**: 每条名言附简短背景故事/出处
- **注意**: 需考据名言真实性（尤其爱因斯坦相关）

### 3.9 FurtherReading（延伸阅读）
- **内容**: 书籍、文章、视频推荐链接
- **格式**: 卡片列表，含标题、简介、外链
- **规范**: `target="_blank" rel="noopener"`

## 四、设计方向

### 4.1 配色方案（三选一，待确认）
- **方案 A — 现代金融**: 深蓝 + 金色，专业感，适合金融类
- **方案 B — 学术白**: 白底 + 黑字 + 单色强调，极简阅读体验
- **方案 C — 友好科普**: 暖色系 + 圆角 + 插画风，亲和力强

### 4.2 技术栈
- Vite + React + TypeScript
- Tailwind CSS（样式）
- Recharts（图表）
- Framer Motion（动画）
- Vitest（测试）

### 4.3 响应式策略
- 桌面端 (≥1024px)：完整布局
- 平板端 (768-1023px)：图表简化
- 手机端 (<768px)：单列布局，图表最小化

## 五、非功能需求

### 5.1 性能
- Lighthouse Performance ≥ 90
- 图表组件懒加载
- 图片使用 WebP 格式

### 5.2 无障碍
- WCAG AA 级别
- 所有交互键盘可达
- 颜色对比度 ≥ 4.5:1
- 支持 `prefers-reduced-motion`

### 5.3 SEO
- Meta 标签 / OG Image
- 结构化数据 (JSON-LD)
