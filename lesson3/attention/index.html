双网络拮抗模型Task-Positive Network ↔ Default Mode Network

大脑两套网络此消彼长：专注网络主动耗能、默认网络省电待机。

何时调用：当你"忍不住打开手机"时，提醒自己——这不是意志薄弱，这是默认网络在生效。

想象你在专心读一本书。30 分钟后你已经累了——这不是矫情，前额叶皮质在主动工作时持续消耗葡萄糖。

你松懈的瞬间，默认模式网络（DMN）接管。它是大脑的"省电模式"——胡思乱想、白日梦、刷手机、记起某条没回的微信。它消耗的能量只有专注的几分之一。

所以分心不是你的错，是物理学。但能不能坚持专注，是设计的事。

Module 02

多巴胺劫持循环

论点：注意力被劫持的不是奖励本身，而是"不确定的预期"。

↑ 试试看，会发生什么

你停下来了。
这是这一整天里你为自己做的最重要的决定。

你追逐的是预期，不是奖励本身。

多巴胺贩卖的是"下一个会更好"的幻觉。

奖励预测误差陷阱Reward Prediction Error Trap

公式：触发 → 渴望 → 不确定反馈 → 更强渴望。不确定性是劫持的终极武器。

何时调用：任何"忍不住再刷一下"的产品（短视频/直播/抽卡/邮箱通知/股票 App），其商业模式都嵌入了这个公式。

赌博机不是一直让你赢钱才上瘾的。它让你有时候赢钱。

不确定的奖励比确定的奖励刺激强 3-5 倍——大脑分泌多巴胺的不是反馈本身，是"反馈即将到来"的预期。

抖音、小红书、邮箱、微信通知、股票 App、淘宝推荐——所有让你"忍不住再看一眼"的东西，都嵌入了这个公式。它们不是在卖内容，是在卖预期。

Module 03

管窥效应

论点：贫穷与忙碌通过"占用工作记忆"来掠夺注意力，强迫人做短视决策。

远方的山家人书架 健身房夕阳长期目标
房贷工作邮件父母身体截止日期

点亮焦虑事件，看你的世界如何被压缩。

贫穷不始于口袋的空虚，
而始于注意力的溃散。

认知带宽税Cognitive Bandwidth Tax · Mullainathan & Shafir, 《Scarcity》

缺钱 / 缺时间不直接致命，但会引发"管窥心态 (Tunneling)"：管内极清晰、管外漆黑，工作记忆被紧急事务占满，无余裕做长期决策。

何时调用：当自己或他人在做"明显短视的决定"时，先检查 Ta 的认知带宽是否被占满，而非苛责"为什么不长远思考"。

《Scarcity》一书的实验：让两组人玩游戏，"富人"每轮有充足时间，"穷人"每轮时间紧张。

"穷人"在自己擅长的关卡上表现更好——管窥让他们极度聚焦。但他们在所有需要长期规划的关卡上表现糟糕。

管窥不是道德缺陷，是认知带宽被占满时的必然反应。当你或别人做出"明显短视"的决定，先问：Ta 的工作记忆里塞了多少紧急事？

心流通道Csikszentmihalyi Flow Channel

心流发生在挑战与技能都处于高位且挑战略微超出舒适区的狭长通道。

何时调用：当你做事感到"焦虑"或"无聊"时，先调整两个旋钮——挑战难度、当前技能——而不是责怪自己"不够自律"。

米哈里·契克森米哈伊用 30 年访问了几千个进入心流的人——艺术家、运动员、外科医生、流水线工人。他发现规律：

心流不是性格，不是天赋，不是运气。是当下任务的挑战和你当前的技能匹配，且挑战略微高于技能。

这意味着你可以主动调出心流：把工作切到合适的难度，把环境清掉分心源，启动一个仪式（音乐、咖啡、固定时间）。心流不会自己来。

## 4. 视觉与交互规范

### 4.1 色板（Apple 浅色 · Light Appearance）

参考 Apple Human Interface Guidelines (Light) 与 apple.com 产品页系统色。

| 用途 | 色值 | 含义 |
|---|---|---|
| 主背景 | `#ffffff` | 纯白（apple.com 主背景） |
| 次级背景 | `#fbfbfd` | 区块/分节背景（苹果官网经典淡灰白） |
| 表面/卡片 | `#f5f5f7` | 卡片底色（System Gray 6 浅色） |
| 边框/分隔 | `rgba(0,0,0,0.08)` | 极淡边框，仅在必要时使用 |
| 主文字 | `#1d1d1f` | 接近纯黑（apple.com 标题色） |
| 次文字 | `#6e6e73` | 中灰（副标题、说明文字） |
| 弱文字 | `#86868b` | 浅灰（注释、metadata） |
| 主强调（链接/CTA） | `#0071e3` | Apple Blue（系统蓝，链接、主按钮） |
| 警示红 | `#ff3b30` | Apple Red（陷阱 / 收割 / 焦虑区） |
| 心流绿 | `#30d158` | Apple Green（心流通道 / 复利投向） |
| 多巴胺紫 | `#bf5af2` | Apple Purple（不确定奖励 / 多巴胺） |
| 暖橙（默认网络） | `#ff9f0a` | Apple Orange（跷跷板分心端） |

**强调色使用克制**：所有彩色都仅出现在「数据/状态/语义」位置，不用于装饰。文字 99% 是 `#1d1d1f` 或 `#6e6e73`。

### 4.2 字体

- 全局字体栈：`-apple-system, BlinkMacSystemFont, "SF Pro Display", "SF Pro Text", "Helvetica Neue", "PingFang SC", "Noto Sans SC", sans-serif`
- 等宽（数字与代码）：`"SF Mono", ui-monospace, Menlo, Consolas, monospace`
- 字重梯度（Apple 风：大字偏轻，小字偏重）：
  - Hero 巨标题：`font-weight: 600`，字号 `clamp(56px, 9vw, 112px)`，`letter-spacing: -0.04em`
  - 章节标题（H2）：`font-weight: 600`，字号 48-64px，`letter-spacing: -0.02em`
  - 模块小标题（H3）：`font-weight: 600`，字号 28-32px
  - 正文：`font-weight: 400`，字号 17-19px，`line-height: 1.47`
  - 金句：`font-weight: 600`，字号 +30%，颜色 `#1d1d1f`，无引号装饰，居中大字呈现
  - 说明/弱文字：`font-weight: 400`，色 `#6e6e73`
- **数字使用 `tabular-nums`**（如倒计时、剩余小时数），避免跳动。

### 4.3 排版与节奏

- 内容最大宽度 `980px`（与 apple.com 产品文章页对齐），居中。
- 模块间垂直留白至少 `120px`（桌面）/ `80px`（移动）。
- 每个模块固定结构：`小标题 → 引文 → 互动区 → 金句 → 思维模型 → 长论证文字`。
- 长论证文字保持**克制**：每段不超过 4 行，避免与互动区视觉打架。
- **零边框为默认**：用空白和背景色对比（`#ffffff` ↔ `#fbfbfd` ↔ `#f5f5f7`）来分隔区块，而不是用线条。
- 圆角统一：卡片 `18px`、按钮 `980px`（pill 圆角，Apple 经典）、输入框 `12px`。
- 阴影极淡或无：仅在浮起卡片时使用 `0 4px 24px rgba(0,0,0,0.06)`。

### 4.4 动效原则

- 所有动效**默认就绪、不抢夺**。仅在用户主动操作时（拖、点、滚）激活。
- 缓动曲线统一使用 Apple 系：`cubic-bezier(0.4, 0, 0.2, 1)`，时长 `300-500ms`。
- 自动循环动画（如多巴胺循环）必须有"暂停 / 我看够了"按钮——本网页主题不允许劫持读者注意力。
- 滚动时章节标题以 Apple 风格的"渐入上移"出现（`translateY(20px) → 0` + `opacity 0 → 1`）。
- 尊重 `prefers-reduced-motion`，关闭所有非必要动画。

## 8. 制作里程碑（顺序执行）

1. **骨架阶段**: HTML 结构 + 7 个模块的占位区 + 全部文字（含金句、思维模型卡、长论证文字）。验收标准——关掉 JS 也能读完整内容。
2. **样式阶段**: 套入 Apple 浅色色板 + 排版规范（SF Pro 字体栈、纯白背景、巨型轻量标题、Apple 系统语义色）。验收标准——视觉气质对标 apple.com 产品页：纯白底、克制留白、零多余边框、单焦点叙事。
3. **交互阶段**: 按模块编号 1→7 逐个加交互。每加完一个，自测移动端降级。
4. **细节阶段**: 顶部封面呼吸光点 + 章节过渡卡片 + 注意力守则卡 + 另存为 PNG。
5. **审阅阶段**: 通读一遍，砍掉任何"装饰但不承载论点"的元素。再次提醒：本网页主题不允许劫持读者注意力。
